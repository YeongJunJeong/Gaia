# Project Gaia: 토양 미생물 파운데이션 모델 오픈소스 로드맵

---

## 프로젝트 비전

**"토양 미생물의 AlphaFold"를 오픈소스로 만든다.**

> *Gaia — 그리스 신화의 대지의 여신. 모든 생명의 토대인 토양, 그 속에 숨겨진 미생물의 언어를 해독하는 프로젝트.*

토양 미생물 군집의 "언어"를 이해하는 파운데이션 모델을 공개 데이터로 사전학습하고, 이를 기반으로 토양 건강 진단, 수확량 예측, 미생물 컨소시엄 설계를 가능하게 하는 오픈 플랫폼을 구축한다.

## Phase 1: 데이터 수집·전처리 (2~4주)

### 1-1. 공개 데이터 수집

**소스 A: MGnify — 분류학적 풍부도 테이블 (1순위)**

```python
# MGnify API로 토양 바이옴 샘플 필터링
# biome_name에 "soil" 포함 샘플 수집
# 출력: 속(genus) 수준 풍부도 테이블
```

목표: 토양 관련 샘플 5,000~15,000개 수집
포함 바이옴: agricultural soil, forest soil, grassland soil, 
            desert soil, permafrost, rhizosphere

**소스 B: NEON — 짝 데이터 포함 (최고 가치)**

```
수집 항목:
- 토양 미생물 메타게놈 시퀀싱 데이터 (DP1.10107.001)
- 토양 화학 데이터: pH, 유기탄소, 총질소 (DP1.10086.001)
- 토양 물리 데이터: 수분, 온도 (DP1.00094.001)
- 기상 데이터: 강수, 기온 (DP1.00006.001)
```

가치: 미생물 데이터 + 환경 데이터가 짝으로 연결된 유일한 대규모 공개 자원

**소스 C: Earth Microbiome Project — 표준화된 프로토콜**

가치: 전 세계에서 동일한 프로토콜로 수집된 데이터 → 배치 효과 최소화

**소스 D: SMAG 카탈로그 — 토양 MAG 참조**

3,304개 토양 메타게놈에서 복원된 40,039개 MAG
→ 기능 유전자 주석용 참조 데이터베이스로 활용

**소스 E: 개별 논문의 보충 데이터**

가뭄 스트레스 데이터셋 (Naylor et al., 623 샘플)
러시아 흑토 메타게놈 (shotgun, 기능 유전자 포함)
농경지 초심층 시퀀싱 (600 샘플, 평균 107Gb/샘플)

### 1-2. 전처리 파이프라인

```
[원시 데이터]
    │
    ▼
[1] 분류체계 통일 ──── GTDB r220 기준으로 매핑
    │                   (연구마다 다른 DB → 하나로 통일)
    │
    ▼
[2] 풍부도 정규화 ──── 옵션 A: TSS (Total Sum Scaling, 상대적 %)
    │                   옵션 B: CLR (Centered Log-Ratio)
    │                   → 두 방법 모두 실험 후 비교
    │
    ▼
[3] 스파시티 필터링 ── 전체 샘플의 0.1% 미만에서만 
    │                   나타나는 속(genus) 제거
    │                   → 노이즈 감소, 어휘 크기 관리
    │
    ▼
[4] 메타데이터 정제 ── 바이옴 분류 통일 (ENVO 온톨로지)
    │                   지리적 위치 표준화
    │                   누락/의심 데이터 플래그
    │
    ▼
[5] 배치 효과 태깅 ── 시퀀싱 플랫폼, DNA 추출 키트,
    │                   분석 파이프라인을 메타데이터로 기록
    │                   → 사후 보정에 활용
    │
    ▼
[6] 코퍼스 변환 ────── MGM 형식: 풍부도 순 정렬 → 토큰화
                        입력 길이: 512 (99.99% 커버)
                        어휘: 토양 관련 속 ~3,000~5,000개
```

### 1-3. 데이터 품질 기준 (Quality Checklist)

| 기준 | 임계값 | 미달 시 조치 |
|------|--------|-------------|
| 총 리드 수 | > 10,000 | 제거 |
| 분류된 속 수 | > 20 | 제거 |
| 메타데이터 완전성 | 바이옴 + 위치 필수 | 없으면 "unknown" 태깅 |
| 상위 1종 점유율 | < 90% | 오염 의심 → 검토 |
| 시퀀싱 플랫폼 정보 | 있어야 함 | 없으면 배치 보정 불가 태깅 |

## Phase 2: 사전학습 (4~6주)

### 2-1. 전략 선택: 처음부터 vs 전이학습

**권장: MGM 가중치에서 시작하여 토양 특화 사전학습 (계속학습, Continual Pre-training)**

```
[MGM 가중치] ──── 26만 전체 바이옴 학습 완료 상태
      │
      ▼
[토양 코퍼스로 추가 학습] ──── 토양 샘플 집중 노출
      │                         → 토양 미생물 간 관계 심화 학습
      ▼
[Gaia v0.1]
```

이유:
- MGM이 이미 "미생물의 일반적 언어"를 학습한 상태
- 토양 데이터만으로 처음부터 학습하면 데이터 부족
- 계속학습으로 토양 특화 패턴을 추가 학습하는 것이 효율적

### 2-2. 모델 아키텍처

```
입력: [속1_토큰] [속2_토큰] ... [속512_토큰]
       (풍부도 순 정렬)

모델: 다층 트랜스포머 디코더
      - 레이어: 6~12 (데이터 규모에 따라 조정)
      - 어텐션 헤드: 8~16
      - 임베딩 차원: 256~512
      - 어휘 크기: ~5,000 (토양 관련 속)

사전학습 과제: 인과적 언어 모델링 (Causal LM)
      → "앞의 미생물들을 보고 다음 미생물을 예측"

추가 사전학습 과제 (선택):
      - 마스킹 복원: 30~50% 마스킹 후 복원
      - 풍부도 예측: 속은 주고 풍부도를 맞추기
```

### 2-3. 학습 설정

```yaml
# 학습 하이퍼파라미터 (초기 설정, 튜닝 필요)
batch_size: 64
learning_rate: 1e-4 (warmup + cosine decay)
warmup_steps: 1000
max_epochs: 50
optimizer: AdamW
weight_decay: 0.01
gradient_clipping: 1.0
fp16: true  # 메모리 절약

# 데이터 분할
train: 85%
validation: 10%
test: 5% (절대 학습에 사용 안 함)
```

### 2-4. 사전학습 품질 검증

학습이 제대로 되고 있는지 확인하는 지표:

**지표 1: 복원 정확도**
- 30% 마스킹 시 복원 코사인 유사도 > 0.85
- 50% 마스킹 시 > 0.75
- MGM의 20% 마스킹 시 0.9 이상 달성을 참조 목표로

**지표 2: 임베딩 군집화**
- UMAP/t-SNE로 샘플 임베딩 시각화
- 같은 바이옴(농경지, 산림, 초원)끼리 군집화되는지 확인
- 실루엣 점수 > 0.5

**지표 3: 분류학적 구조 학습**
- 속 임베딩에서 세균/곰팡이/고세균이 분리되는지
- 계통학적으로 가까운 속들이 임베딩 공간에서도 가까운지

**지표 4: 생태학적 의미**
- 알려진 공생 관계 미생물이 임베딩에서 가까운지
  (예: 균근균 ↔ 식물 성장 촉진 세균)
- 알려진 길항 관계가 반영되는지

### 2-5. 산출물

- `gaia-v0.1.pt`: 사전학습 완료 가중치
- 학습 로그 및 지표 시각화
- 모델 카드 (Model Card): 학습 데이터, 설정, 한계 명시

## Phase 3: 벤치마크 구축 (2~3주)

### 3-1. 벤치마크 과제 정의

사전학습 모델의 성능을 객관적으로 비교할 수 있는 표준 벤치마크를 구축합니다.
(단백질 분야의 CASP, 언어 분야의 GLUE에 해당)

**과제 1: 토양 바이옴 분류 (Source Tracking)**
- 입력: 미생물 풍부도 프로필
- 출력: 바이옴 유형 (농경지/산림/초원/사막/습지)
- 지표: ROC-AUC, F1 score
- 데이터: MGnify에서 바이옴 레이블이 있는 토양 샘플

**과제 2: 토양 화학 특성 예측**
- 입력: 미생물 풍부도 프로필
- 출력: pH, 유기탄소, 총질소 (연속값)
- 지표: R², RMSE
- 데이터: NEON (미생물 + 화학 짝 데이터)

**과제 3: 경운 방식 분류**
- 입력: 미생물 풍부도 프로필
- 출력: 무경운/최소경운/관행경운
- 지표: Accuracy, Kappa
- 데이터: 기존 논문에서 수집

**과제 4: 가뭄 스트레스 탐지**
- 입력: 미생물 풍부도 프로필
- 출력: 정상/가뭄 스트레스 (이진 분류)
- 지표: Accuracy, F1
- 데이터: Naylor et al. 데이터셋 (623 샘플, 공개)

**과제 5: 미생물 풍부도 복원 (사전학습 품질)**
- 입력: 20%/30%/50% 마스킹된 프로필
- 출력: 원래 프로필 복원
- 지표: 코사인 유사도, MAE
- 데이터: 테스트셋

### 3-2. 베이스라인 모델

각 과제에서 Gaia과 비교할 기준 모델:

| 모델 | 유형 | 설명 |
|------|------|------|
| Random Forest | 전통 ML | 마이크로바이옴 분야 표준 |
| XGBoost | 전통 ML | 부스팅 기반 |
| SVM (L2) | 전통 ML | 정규화 포함 |
| MGM (원본) | 파운데이션 모델 | 전체 바이옴 사전학습 |
| MGM (미세조정 없음) | 제로샷 | 사전학습만으로 성능 확인 |
| Gaia (미세조정 없음) | 제로샷 | 토양 특화 사전학습 효과 확인 |
| Gaia (미세조정) | 전이학습 | 최종 성능 |

### 3-3. 산출물

## Phase 4: 미세조정 및 응용 (4~8주)

### 4-1. 간접 짝 데이터를 활용한 미세조정

자체 수집 데이터 없이 공개 데이터에서 추출할 수 있는 짝 데이터:

```
[미세조정 과제 1] 토양 화학 예측
├── 데이터: NEON (미생물 + pH/C/N)
├── 방법: 회귀 헤드 추가 → MSE 손실
└── 의의: "미생물 → 토양 상태" 기본 관계 학습

[미세조정 과제 2] 기능 유전자 풍부도 예측
├── 데이터: 16S + 샷건 시퀀싱이 모두 있는 샘플
├── 방법: 16S 입력 → 기능 유전자 풍부도 예측
└── 의의: 저비용(16S)으로 고비용(샷건) 결과를 추정

[미세조정 과제 3] 가뭄 스트레스 분류
├── 데이터: Naylor et al. (623 샘플)
├── 방법: 분류 헤드 추가 → 교차 엔트로피
└── 의의: 환경 스트레스 탐지 능력 검증

[미세조정 과제 4] 토양 유형별 핵심 미생물 식별
├── 데이터: 벤치마크 과제 1 데이터
├── 방법: attention weight 분석 → SHAP 비교
```
